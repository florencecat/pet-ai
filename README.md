# Спутник питомца 🐾

**Главный помощник в уходе за домашними питомцами.** Мобильное приложение, которое помогает владельцам не забывать давать лекарства, планировать визиты к ветеринару, вовремя делать прививки и обработки, следить за весом, питанием и настроением питомца — а ещё задавать вопросы ИИ-ассистенту по уходу.

> Flutter-приложение (Android / iOS). Локально-ориентированное: основной функционал работает без интернета, облако и ИИ подключаются опционально.

---

## Возможности

### Питомцы и профили
- Несколько профилей питомцев с быстрым переключением
- Поддержка видов и пород, фото профиля (камера / галерея с обрезкой)
- Персональная цветовая палитра под каждого питомца
- День рождения с обратным отсчётом и расчётом «человеческого» возраста

### Здоровье и трекинг
- **Оценка здоровья** (норма / внимание / критично) на основе умного анализа:
  - тренд веса, аномалии аппетита, просроченные прививки и события
- **Вес** — графики и история
- **Питание** — время приёма, аппетит, вес порции
- **Настроение** — с сортировкой по дате и времени суток
- **Заметки** — по симптомам с готовыми тегами и голосовым вводом
- **Препараты** — напоминания, несколько приёмов в день, иконка и цвет в iOS-стиле
- **Обработки** — прививки, паразиты и т.п. с автосозданием следующего события
- **Документы** — хранилище файлов по категориям (прививки, страховка…), загрузка с камеры или из файлов

### События и календарь
- Создание и управление событиями, категории
- Повторяющиеся события с настраиваемым правилом (выбор дней недели) и датой окончания курса
- События на весь день (время указывать необязательно)
- Отметка выполнения по каждому повторению
- Привязка одного события к нескольким питомцам, режим «все питомцы»
- Подсветка и приоритизация просроченных событий

### ИИ-ассистент
- Чат с помощником по уходу (бэкенд на GigaChat)
- Интерактивные треды: переключение между диалогами с сохранением контекста
- Прикрепление сущностей (препарат, заметка, прививка, событие) к сообщению — уходят в запрос в JSON
- Реальный индикатор доступности бота (`/health`), корректная обработка ошибок сервера

### Облако и надёжность
- Синхронизация с облаком через **PocketBase** (авторизация, восстановление данных из облака при онбординге)
- Отправка ошибок и стектрейсов в PocketBase (переключатель в настройках)
- Локальные уведомления-напоминания

### Интерфейс
- «Стеклянный» (glass) UI, плавающая навигация, скелетоны загрузки
- Анимации и переходы, единый стиль выезжающих листов (draggable sheets)
- Управление внешним видом через `AppearanceController`

## Стек и архитектура

- **Flutter** (Dart SDK `^3.9.2`), Material + Cupertino, локализация (`ru`, `en`)
- **Состояние:** `provider` (ChangeNotifier) + `get_it` (синглтоны сервисов)
- **Локальное хранилище:** `hive` (чат), `shared_preferences` (настройки), файловая система (файлы / аватары)
- **Облако:** `pocketbase` — пользователи, питомцы, события, `error_reports` и др.
- **Сеть:** `http` поверх `IOClient` с доверенным корневым сертификатом (Russian Trusted Root CA, см. `assets/certs/`)
- **Графики:** `fl_chart`; **календарь:** `table_calendar`
- **Прочее:** `image_picker` / `image_cropper`, `speech_to_text`, `flutter_local_notifications`, `flutter_svg`, `google_fonts`, FontAwesome

### Бэкенды

| Назначение | URL | Эндпоинты |
|---|---|---|
| API / данные | `https://api.pet-sputnik.ru` | PocketBase-коллекции (`users`, `pets`, `events`, `error_reports`, …) |

## Запуск

### Требования
- [Flutter](https://docs.flutter.dev/get-started/install) (канал `stable`)
- Android Studio / Xcode для целевых платформ

### Шаги

```bash
cd pet_ai

# 1. Зависимости
flutter pub get

# 2. Кодогенерация Hive-адаптеров (ai_service.g.dart)
dart run build_runner build --delete-conflicting-outputs

# 3. Запуск на устройстве/эмуляторе
flutter run
```

### Конфигурация (`.env`)

Приложение читает `pet_ai/.env` (подключён как asset). Скопируйте шаблон и заполните значения:

```bash
cp .env.example .env
```

```env
GIGACHAT_KEY=<ваш_ключ_авторизации_GigaChat>
```

> ⚠️ `.env` не коммитится (`.gitignore`) — храните секреты локально. Для ИИ-функций нужен валидный `GIGACHAT_KEY`; без него остальной функционал приложения работает.

## Сборка

```bash
# Android
flutter build apk --release      # или: flutter build appbundle --release

# iOS
flutter build ipa --release
```

Иконки приложения генерируются через `flutter_launcher_icons`:

```bash
dart run flutter_launcher_icons
```

CI собирает iOS-сборку через Codemagic (workflow `ios-workflow` в [`codemagic.yaml`](codemagic.yaml)).
